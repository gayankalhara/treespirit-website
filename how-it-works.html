How it works?

The proposed system consists of Listening Posts, Sound Processors and a Base Station. The Listening Post continually listens and transmits the sounds of the forest to the designated Sound Processor. Each Sound Processor is allocated with three Listening Posts. The Sound Processor will process the sound to identify any signals that could potentially be chainsaw noise. If it does detect such a sound, it will locate the origin of the sound. Then this location information is transmitted to the base station. The base station uploads such received data to the Cloud Application Programming Interface (API) which makes the data readily available through the associated web application and Short Message Service (SMS) gateway.

The Listening Posts are based on the Arduino Uno device while the Sound Processors and the Base station are based on the Raspberry Pi 3 Model B device. All devices are powered by solar panels with battery packs as backup power systems.

Listening Posts

The Listening Post is the most elementary component of the system. It is a system that consists of a Digital MEMS Microphone, A Real Time Clock (RTC), A high speed – low range communication chip and a Temperature + Humidity Sensor. Its main purpose is to transmit live sounds from its surroundings to the allocated Sound Processor for 20 seconds every minute along with the Temperature and Humidity data it has collected at the moment. In fact, it is correct to identify it as a Sensor node which is continuously transmitting data which can be then interpreted by the higher components of the system.

Sound Processor

The Sound Processor contains Software modules to analyse the sounds to identify chainsaw sounds and if such sounds are identified, estimate the origin of the such sounds. These two modules will be identified as “Sound Recognition Module” and “Sound Localization Module” here on respectively. It also has hardware modules to communicate with the Listening posts assigned to it as well as the Base Station. Upon the receival of a sound, the Sound Processor reacts in two ways depending on the outcome of the analysis in the Sound Recognition Module.

  • Case 1 – Sound is identified as a chainsaw sound
  • Case 2 – Sound cannot be identified as a chainsaw sound

In the case 1, Once the received signal is identified as having a strong probability to be a chainsaw sound, the Sound Processor then activates the Sound Localization module it contains. The sound localization will be carried out using a method called Multilateration. Once the localization is complete, the Sound Processor will then transmit an alert indicating possible logging operation at the given coordinates/ approximate area in the forest over an ISM Band Radio Link to the Base Station. This operation happens real-time.

In the case 2, the Sound Processor will ignore the Sound and avoid taking further actions.

Base Station

Once an alert is reached to the base station via a Sound Processor, base station uploads the data to a cloud server.

Web Application

The Web Application aggregates and displays all the locations and alerts transmitted by a base station device in an interactive map. Further, it provides features to view and analyze historical data as well as information about Clusters.

About TreeSpirit

TreeSpirit is a Forest Monitoring System based on Wireless Sensor Network (WSN) and Chainsaw Noise Identification using a Convolutional Neural Network.
